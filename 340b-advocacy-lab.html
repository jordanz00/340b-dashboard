340B Advocacy Lab — Pennsylvania

Verified static data, DataLayer API, and print export. No CDN scripts. Read the comments in modules/advocacy-lab.js.

1. Pennsylvania hospital locations

Dots are geocoded positions from the HAP Resource Center participating-hospital list (see pa-340b-hospitals.js meta). This is not per-hospital 340B savings data.

2. Headline metrics (bar chart)

Bars are built from DataLayer.getKPIs(). National community benefit ($7.95B) is a separate metric — see footnote.

3. Story submission (structured)

Fields map toward fact_story_submission; extended columns can be added in Gold. Submissions store in sessionStorage until an API is configured.

4. Advocacy report export

Builds a one-page HAP brief using the same print system as the main dashboard (print-view.css, aligned to the leave-behind / 340B_032726.pdf spec). Uses window.print() — choose Save as PDF. Wait for the PA map to finish loading before printing so the snapshot includes the map.

5. AI helper stubs

AIHelpers runs offline today; swap in an endpoint later without changing call sites.

6. Sources & methodology

  • Hospital dots: data/pa-districts/pa-340b-hospitals.js — HAP Resource Center list, revision 2022-05-27; coordinates via OpenStreetMap Nominatim (see each row source).
  • KPI values: state-data.js (HAP_STATIC_METRICS, CONFIG) and computed state-law counts; warehouse path uses fact_dashboard_kpi.
  • Citations: powerbi/metric-registry.json and CONFIG.copy.sourceSummary in state-data.js.
  • Not in this repo: Facility-level 340B savings — load from Gold when IT publishes; do not fabricate for charts.

Full field dictionary: docs/DATA-DICTIONARY.md (Advocacy Lab section).
